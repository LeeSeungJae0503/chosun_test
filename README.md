# 리눅스 프로세스 명령어 정리

## 1. `top` — 실시간 프로세스 모니터링

- CPU·메모리 사용률, 실행 중인 프로세스를 실시간으로 갱신하여 보여주는 도구
- Windows의 작업 관리자와 유사한 역할

### 주요 특징

- 프로세스 상태를 초 단위로 업데이트
- CPU 사용률, Load average, 메모리 사용량 등을 상단에서 확인
- 하단에는 PID, USER, %CPU, %MEM 등 리스트 표시

### 자주 사용하는 키

- `q` : 종료
- `P` : CPU 사용률 기준 정렬
- `M` : 메모리 기준 정렬
- `T` : 실행 시간 기준 정렬
- `k` : 프로세스 종료(kill)
- `1` : 멀티코어 CPU 개별 사용량 보기

### 자주 쓰는 실행 옵션

- `top -u 사용자` : 특정 사용자 프로세스만 보기
- `top -n 5` : 5번 갱신 후 종료

## 2. `ps` — 프로세스 스냅샷 조회

- 호출 시점의 프로세스 정보를 한 번만 출력
- 스크립트나 특정 프로세스 존재 여부 점검에 흔히 사용

### 자주 사용하는 명령 조합

- `ps aux` : BSD 스타일 전체 프로세스 출력
- `ps -ef` : System V 스타일 전체 프로세스 출력
- `ps -p PID` : 특정 PID 보기
- `ps -u USER` : 특정 사용자 프로세스 보기
- `ps aux | grep python` : 특정 키워드 포함 프로세스 찾기

### 주요 출력 컬럼

- `PID` : 프로세스 ID
- `%CPU`, `%MEM` : CPU/메모리 사용률
- `STAT` : 현재 상태(R: 실행, S: 대기, T: 중지, Z: 좀비 등)
- `START`, `TIME` : 시작 시각과 누적 CPU 사용 시간
- `COMMAND` : 실행된 명령어


## 3. `jobs` — 현재 셸의 백그라운드 작업 관리

- 현재 터미널 세션 에서 실행하거나 중지한 작업 목록 확인
- 시스템 전체가 아니라 “현재 셸에서 실행한 job”만 표시한다는 점이 중요

### 사용 흐름

1. 명령 뒤에 `&` → 백그라운드 실행
2. `Ctrl + Z` → 실행 중인 프로세스 일시 중지
3. `bg %1` → 중지된 작업을 백그라운드에서 재개
4. `fg %1` → 특정 작업을 포그라운드로 가져오기
5. `jobs` → 현재 job 목록 확인

### jobs 출력 예시

```
[1]+  Running   python script.py &
[2]-  Stopped   vim test.txt
```

- `+` : 기본적으로 fg/bg 적용되는 job
- `Stopped` : Ctrl+Z 등으로 멈춘 상태

## 4. `kill` — 프로세스 종료

- 특정 프로세스에 signal(신호) 를 보내기 위한 명령어
- 가장 흔한 용도 = 프로세스 종료

### 사용법

- `kill PID` : SIGTERM(15) → 정상 종료
- `kill -9 PID` : SIGKILL → 강제 종료
- `kill -HUP PID` : 데몬 재시작 용도로 자주 사용되는 신호

### 주요 Signal 종류

| Signal  | 번호 | 설명                                 |
| ------- | ---- | ------------------------------------ |
| SIGTERM | 15   | 정상 종료 요청(정리 후 종료)         |
| SIGKILL | 9    | 무조건 종료(회복 불가)               |
| SIGHUP  | 1    | 연결 끊김, 데몬 재시작 트리거로 사용 |
| SIGINT  | 2    | Ctrl + C                             |
| SIGSTOP | 19   | 강제 중지(잡을 수 없음)              |
| SIGCONT | 18   | 중지된 프로세스 재개                 |

### 프로세스 찾고 종료하는 예시

- `ps aux | grep python` → PID 확인
- `kill PID` → 종료
- 안 끝나면 → `kill -9 PID`

# 네 가지 명령어  정리

| 명령어   | 역할                           |
| -------- | ------------------------------ |
| **top**  | 실시간 모니터링                |
| **ps**   | 스냅샷 조회                    |
| **jobs** | 현재 셸의 백그라운드 작업 관리 |
| **kill** | 프로세스 종료/제어             |

### 작동 영역

- `top`, `ps` → 시스템 전체 프로세스
- `jobs` → **현재 터미널/셸**에 한정
- `kill` → 특정 PID에 신호 전달
